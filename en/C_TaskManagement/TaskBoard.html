Task Board

The tiles that are displayed on the main view of the task board, contain a brief overview over all upcoming tasks. The tasks were gathered after their respective task types.

Click on a tile to get more detailed information about all upcoming tasks. The detailed view is displayed next to the tiles in the CDH2 main pane. Its header section contains all information shared by all tasks of the respective task type, like 'Assignees', 'Due date', 'Priority', etc. The section underneath the header displays a list with all records within that task related to a certain assignee.

The tasks can be filtered by the drop-down fields in the header to get smaller portions of the list reflecting special perspectives or purposes.

The available filter options are:

  • Assignees
  • Due date (Tasks according their due date)
  • Priority (Very high, High, Normal, Low, Very low)
  • MyTasks (tasks that are assigned to you)
  • MyTeamTasks (tasks that are assigned to your team)
  • All tasks of a certain type (User defined tasks as well as automatically generated tasks like: [RAW, VAL, MATCH, MERGE, ReVAL, ])

Each line of the list, task lines as well as record lines, are equipped with a 'Review' button. Click on that button to switch into the detailed task view or the respective record view.

How to create a new task

Creating a new task always means to create tasks manually without being necessarily caused by an error in the validation, matching or merging phase.

Fill in the required input fields. Input fields marked with '*' are mandatory fields. Once you created a new task, it will be displayed in the task overview.

  • Click on 'Create task' to confirm your entries and to create a new task.
  • Click 'Cancel' to abort.

Example:

The following input fields are available:

Name:

Your name. After a news task is created you will find your name in the field 'Reporter'.

Description:

Short description of the issue.

Record:

ID of the record that caused the issue.

Due date:

Due date of the deadline for resolving the task. This is a drop-down field. You can select the date from a calendar.

Priority:

Priority for accomplishing a task. The available levels are: Very high, high, low and very low. Default level is 'normal'.

Assignees:

Name of the person that is responsible for accomplishing the task.

Select a name from the drop-down field. 'Unassigned' is always predefined (Default setting).

Check for duplicate record:

A record that didn't pass the validation procedure can be checked for duplicates anyway. This check box is inactive as long as no record was selected that matches this criterion. Set a check mark into the check box for running a matching process.

How to create a new task (bulk)

Creating a new task always means to create tasks manually without being necessarily caused by an error in the validation, matching or merging phase.

Creating multiple new tasks always means to create tasks manually for a larger set of records. This can be necessary in case that the same issue occurs in a bulk of records.

Fill in the required input fields. Input fields marked with '*' are mandatory fields. Once you created a new task, it will be displayed in the task overview.

  • Click on 'Create task' to confirm your entries and to create a new task.
  • Click 'Cancel' to abort.

Example:

The following input fields are available:

Name:

Your name. After a news task is created you will find your name in the field 'Reporter'.

Description:

Short description of the issue.

Record:

IDs of the records that caused the issue.

Due date:

Due date of the deadline for resolving the task. This is a drop-down field. You can select the date from a calendar.

Priority:

You can define a priority level for the new task. The following priority levels are available: Very high, High, Normal, Low, and Very low. Select one from the drop-down field. Normal is always predefined (Default setting).

Assignees:

Assign the new task to a person that is in charge to deal with it. Select a name from the drop-down field. 'Unassigned' is always predefined (Default setting).
Task post processing - How to review a problem

The task view provides several functions for the post processing procedure. Details are described in the "Task overview" section.

Underneath the header section with the current task settings the record view is displayed. It provides a content view on the data level of the record.

In some cases the collapsed task views of different tasks that are related to the displayed record.

  • Click on the 'Expand' button to open the respective task view. You can fix all issues on that record by accomplishing all related tasks.
  • Click on the 'Record action' button underneath the record ID to open the 'data' view of the record and to fix the problem.
  • Click on 'Delete task' for deletion
  • Click on 'Mark as resolved' to resolve the task for further processing, after you fixed the issues within the data of that record.

Due date:

Date, by which the task has to be accomplished. The format of the entry can be defined by the user.

Status:

Status of processing progress of the task, resp. the stage of completion of the task. Possible status designations will be open or resolved.

Priority:

Priority for accomplishing a task. The available levels are: Very high, high, low and very low. Default level is 'normal'.

Assignee:

Name of the person that is responsible for accomplishing the task.

Type:

Category to which the task belongs. Possible types of tasks are: Validation tasks, matching tasks, merge tasks and manual tasks.

Reporter:

Name of the person that reported the cause of the task.

Reported at:

Date when the event that caused the task was reported. The format of the entry can be defined by the user.

Records:

Record ID(s) of the record(s) that is (are) related to the displayed task.

Task post processing - How to use the merge assistant

The task view provides several functions for the post processing workflow. Details are described in the "Task overview" section.

Underneath the header section with the current task settings the record view is displayed. It provides several columns with a content view on record level of all records that are supposed be merged or kept separated.

  • Click on the 'Keep separate' button underneath the 'Merge assistant' header, if you don't want to merge the displayed records.
  • Click on the 'Merge' button to merge the displayed records.
